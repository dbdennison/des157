Needfinding

  1. Provide a brief such that someone visiting the site without prior expectations or context can begin processing the content in a useful way.
  2. Set expectations on what the site can provide or do.
  3. Use consistent information hierarchy patterns and style.
  4. Guide the navigation process – do not make the user guess if we can know their needs.
  5. Keep the interaction loop small – avoid deep heirarchies.
  6. Every section should have a clearly styled call to action.
  7. Use plain english.
  8. Minimize the amount of information presented - embrace skimming.
  9. Follow web conventions to minimize time needed to understand pages.
  10. Do not provide conflicting presentation on the same page.
